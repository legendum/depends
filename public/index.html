Lightweight dependency state tracking. Push state in, get webhooks out.

What is it?

depends.cc tracks the state of things and what they depend on. Each node is a traffic light:

green yellow red

When a node's state changes, depends.cc computes the effective state of everything downstream. If your database goes red, every service that depends on it is effectively red too — automatically, without updating each one.

● database — disk full on /var/data
   ● api-server (effective: red)
      ● worker (effective: red)
● auth-service

How it works

  • Push state — services report their own health via a single PUT. No polling, no agents.
  • Dependency graph — define what depends on what. Effective state propagates automatically.
  • Webhooks — get notified when things change. HMAC-signed, with retries.
  • Reason & solution — attach context: why is this red, and what should I do?

One curl call to set state:

curl -X PUT https://api.depends.cc/v1/state/myproject/database/red \
  -H "Authorization: Bearer $DEPENDS_TOKEN" \
  -H "X-Depends-Reason: disk full on /var/data" \
  -H "X-Depends-Solution: Clear /var/log, expand volume"

Define your graph as YAML, commit it to source control:

namespace: myproject

nodes:
  database:
    label: PostgreSQL Primary

  api-server:
    label: API Server
    depends_on:
      - database

  worker:
    label: Background Worker
    depends_on:
      - api-server

Get started

Sign up to get a token, then create a namespace:

# Sign up (returns a token — save it, shown only once)
curl -s -X POST https://api.depends.cc/v1/signup

# Create a namespace (authenticated)
curl -s -X POST https://api.depends.cc/v1/namespaces \
  -H "Authorization: Bearer $DEPENDS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"id": "myproject"}'

Or use the CLI:

# Install
curl -fsSL https://depends.cc/install.sh | sh

# Sign up and configure
depends signup
export DEPENDS_TOKEN="dep_..."

# Define and push your graph
depends init                                # scaffold depends.yml
depends push                                # sync to depends.cc (auto-creates namespace)
depends status                              # see what's green, yellow, red
depends set myproject/db red --reason "disk full"
depends graph                               # print dependency tree
depends diff                                # preview changes before pushing
depends delete                              # remove a namespace and all its data
API-first, no SDK required.

Any language, any platform. If it can make an HTTP request, it can use depends.cc.
